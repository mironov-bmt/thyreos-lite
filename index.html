// Как это работает
01 / КЛЮЧИ
Отправитель генерирует RSA-4096 + ECDSA P-256. Получатель генерирует RSA-4096. Публичные ключи обмениваются.
02 / ШИФРОВАНИЕ
Генерируется AES-256-GCM ключ, файл шифруется, AES-ключ шифруется RSA-OAEP (публичный ключ получателя), файл подписывается ECDSA.
03 / ПРОВЕРКА
При дешифровке автоматически проверяется ECDSA-подпись отправителя. Если подпись неверна — файл был подменён.
04 / ДЕШИФРОВАНИЕ
RSA-OAEP расшифровывает AES-ключ (приватный ключ получателя), AES-GCM расшифровывает файл. Auth-tag проверяет целостность.
